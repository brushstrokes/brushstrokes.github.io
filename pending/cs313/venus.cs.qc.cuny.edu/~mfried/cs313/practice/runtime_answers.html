Runtime questions

Give a Θ (big theta) estimate for each function.
a. f(n) = 230
Θ(1). Even though it's a large number, it's still a constant.
b. f(n) = n3 + 3n2
Θ(n3). Largest exponent rule. The term with the larger exponent increases at a faster rate and is the most significant term.
c. f(n) = n + logn
Θ(n). For any constant k > 0, nk increases at a faster rate than logn, so n is the most significant term.
d. f(n) = (n2 + n)(n - 1)
Θ(n3). Product rule. (n2 + n) = Θ(n2) and (n - 1) = Θ(n), so the product is Θ(n2 * n) = Θ(n3).

True or false

a. sqrt(n) = O(logn).
False. sqrt(n) = n0.5. For any constant k, nk increases at a faster rate than logn.
b. If f(n) = O(g(n)), then g(n) = O(f(n)).
False. For example, suppose f(n) = n and g(n) = n2.
c. If f(n) = Θ(g(n)), then f(n) = O(g(n)).
True. By definition of big theta, c1g(n) ≤ f(n) ≤ c2g(n). Therefore f(n) ≤ c2g(n).
d. If an algorithm's worst-case runtime is O(n2), it is possible that it takes O(n) time on every input.
True. Big O is just an upper bound. We know the runtime ≤ cn2, but it may also be ≤ cn.
e. If the runtimes of two algorithms are Θ(n2) and Θ(n) (best, worst, and average case), the Θ(n) algorithm definitely runs faster (i.e. takes less time to run) on every input.
False. We don't know the constants. For example, suppose the Θ(n2) algorithm has a runtime of n2, and the Θ(n) algorithm has a runtime of 100n. For small values of n, 100n > n2.
f. 2n+1 = O(2n).
True. 2n+1 = 2 * 2n = O(2n).

What is the runtime of each method? Give your answer in Θ notation as a function of n. 
int foo1(int n) {
   int x = 0;
   for (int i = 0; i < n; ++i)
      for (int j = 0; j < i; ++j)
         ++x;
   return x;
}
Θ(n2).  Each basic operation takes O(1) time.
The inner loop repeats, on average, about n/2 times.  The outer loop repeats n times.
The total number of operations in the loop is about n2/2.

int foo2(int n) {
   int x = 0;
   while (n > 0) {
      n = n / 2;
      ++x;
   }
   return x;
}
Θ(logn).  n is repeatedly divided by 2, so the number of iterations is about logn.

Questions to talk about in class

What if you have a really fast computer? Is an Θ(n2) algorithm on a fast computer faster than an Θ(nlogn) algorithm on a slow computer?
Changing the computer speed is like changing a constant. No matter what the constants are, the Θ(nlogn) algorithm is faster when n is large enough.
