Linear search (average-case)

Θ(n)

Binary search (worst-case)
Θ(logn)

Multiple choice
Which search algorithm should you use to find a number in an unsorted array?
A. Linear search
B. Binary search
C. Interpolation search
Linear search. For binary search and interpolation search, the array must be sorted.

Suppose we modify binary search. Every time we look at the middle element, we also look at the elements immediately to the left and right. If we get lucky, we can end the search early. How does this strategy affect the worst-case runtime?
A. Faster
B. Slower (but still the same big O)
C. No difference
Slower (but still the same big O). The range is still roughly halved in each iteration, but we perform 3 times as many comparisons, so it is slower by a constant factor.

Binary search is not very useful for searching linked lists. Explain why.
In a linked list, we cannot quickly access the middle element.

Questions to talk about in class

Write a method
boolean perfectSquare(int n)
that returns true if n is a perfect square. You may not use Math.sqrt and the runtime must be O(logn).
Binary search to try to find a number whose square is n. Initialize low = 0 and high = n. Note that mid*mid may overflow, so you can avoid this by using a long.

You want to look up the word "binary" in a dictionary as fast as possible (a book, not an online dictionary). How would you do it?
Interpolation search because we know the approximate distribution of English words.
