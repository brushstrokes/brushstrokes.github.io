Which ADT can be described as last-in-first-out?
Stack (in a stack, the most recently added element is removed)

Which ADT can be described as first-in-first-out?
Queue (in a queue, the earliest added element is removed)

Questions to talk about in class

Properly matched parentheses
Write a method
boolean valid(String s)
that returns true if the string consists only of properly matched parentheses and brackets. Every ( must be closed by ) and every [ must be closed by ]. They must be closed in the correct order, and there cannot be extra ) or ] that do not close anything.

Examples:
Valid: ()[]
Valid: ([])
Invalid: (][)
Invalid: ())

Algorithm: Initialize an empty stack. Read each character from left to right.
  • For each character
    • If it's a ( or [, push it into a stack.
    • If it's a ) or ]
      • If it matches the character on top of the stack, pop a character from the stack.
      • If it does not match, or if the stack is empty, return false.
  • After reading every character, if the stack is empty return true, otherwise return false.
Expression evaluation
A fully parenthesized arithmetic expression is either a number, or in the form
( fully_parenthesized_arithmetic_expression operator fully_parenthesized_arithmetic_expression )

Example:
( 1 + ( ( 2 + 3 ) * ( 4 * 5 ) ) )
Expressions like 1 + 2 * 3, which rely on precedence, are not allowed.

Explain an algorithm to evaluate such an expression.

Dijktra's two-stack algorithm: Use two stacks, one for operands (numbers) and one for operators. Read each token from left to right (a "token" can be a left or right parenthesis, an operator, or an operand).
  • If you see an operand, push it onto the operand stack.
  • If you see an operator, push it onto the operator stack.
  • If you see a left parenthesis, ignore it.
  • If you see a right parenthesis, pop an operator and the correct number of operands, perform the calculation, and push the result onto the operand stack.
After finishing, the value of the expression will be on the operand stack.
