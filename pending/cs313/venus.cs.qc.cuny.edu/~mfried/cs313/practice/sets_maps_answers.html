Sets and maps questions

Which Java class would you use to store a collection of unique elements, if you want to be able to insert, delete, and search as fast as possible?
HashSet (TreeSet is slower, but would allow you to iterate through the collection in increasing order)

For the questions below, sorting is not allowed because it would be too slow.

Write a method
boolean containsDuplicate(String[] a)
that returns true if array a contains at least one element that appears more than once. The expected runtime must be O(n) where n is the array length.
Full answer (this answer is for an int array, so you'll have to change the type): DuplicateDetection

Write a method
boolean disjoint(String[] a, String[] b)
that returns true if array a and array b share no elements in common. The expected runtime must be O(m+n) where m and n are the array lengths.
Add all elements of array a to a HashSet. For each element of array b, check if it's in the set, and return the appropriate boolean value.

Write a method
int zeroSum(int[] a)
that checks if array a contains two distinct numbers that sum to zero. If so, return the index of the first number. If not, return -1. The expected runtime must be O(n) where n is the array length.
Use a HashMap (the key is the element, the value is its index). For each element of the array, check if it's negation is a key in the map. If it is, return its index. Then add the pair (element, index) to the map.
