Explain why some implementations of quicksort use insertion sort as a subroutine.
Insertion sort is used to sort small subarrays because insertion sort is faster for small arrays.

Explain how to modify quicksort to guarantee that for all possible inputs, it is very unlikely to perform poorly (there may be multiple correct answers).
Answer 1: Choose random pivots.
Answer 2: Randomly shuffle the array before starting.
Answer 3: Use introsort (start with quicksort but switch to heapsort if the recursion depth grows too large).

Explain what problem the quickselect algorithm solves.
Quickselect finds the kth smallest element of an unsorted array.

Nuts and bolts
You are given n nuts and n bolts. Every nut matches exactly one bolt. You can compare a nut and a bolt, and determine if they match, if the nut is too big, or if the nut is too small. You cannot directly compare two nuts or two bolts. How do you match every nut to a bolt in as few comparisons as possible?
Choose a nut as a pivot. Compare each bolt to a nut to find its match and partition the bolts. Then use the matching bolt as a pivot to partition the nuts. Then recursively sort the two halves. The number of comparisons is about twice as many as quicksort, so it is O(nlogn).

Simultaneous minimum and maximum
We can find the minimum of an unsorted array with n-1 comparisons. How many comparisons does it take to find both the minimum and the maximum? We can do it with 2n-2 comparisons by first finding the minimum, then the maximum, but can we do better?
If n is odd, initialized smallest and largest to the first element.
If n is even, begin by comparing the first two elements to initialize smallest and largest.
For each remaining pair of elements, compare them to each other, then compare the smaller to smallest and the larger to largest.
We do 3 comparisons for every 2 elements, so the number of comparisons is approximately 3n/2.
