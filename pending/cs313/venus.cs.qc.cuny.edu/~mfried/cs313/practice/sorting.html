Write a method
void quicksort(int[] a, int low, int high)
that sorts the elements from index low to index high (inclusive), using the quicksort algorithm. You may assume the existence of a method
int partition(int[] a, int low, int high)
that partitions the array and returns the index of the pivot. All smaller elements will be to the left and all large elements will be to the right. You do not have to write the partition method.

Write a method
void mergesort(int[] a, int low, int high)
that sorts the elements from index low to index high (inclusive), using the mergesort algorithm. You may assume the existence of a method
void merge(int[] a, int low, int mid, int high)
that merges two sorted ranges, a[low..mid] and a[mid+1..high], into one sorted range a[low..high]. You do not have to write the merge method.

Questions to talk about in class

You are given a 5-element integer array, and you want to sort it as fast as possible. How would you do it? If you already know some sorting algorithms, try to forget them. Imagine you are trying to solve this problem for the first time.

What if you want to sort 5 playing cards by hand?

Nuts and bolts
You are given n nuts and n bolts. Every nut matches exactly one bolt. You can compare a nut and a bolt, and determine if they match, if the nut is too big, or if the nut is too small. You cannot directly compare two nuts or two bolts. How do you match every nut to a bolt in as few comparisons as possible?

Simultaneous minimum and maximum
We can find the minimum of an unsorted array with n-1 comparisons. How many comparisons does it take to find both the minimum and the maximum? We can do it with 2n-2 comparisons by first finding the minimum, then the maximum, but can we do better?
