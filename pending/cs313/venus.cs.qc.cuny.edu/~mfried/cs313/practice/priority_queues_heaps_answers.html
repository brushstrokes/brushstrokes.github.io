Questions to talk about in class

How would you find the kth smallest element of an n-element min heap? We can do it in O(klogn) time by calling removeMin() k times, but can we do better?
You can find the kth smallest element in O(klogk) time.
  • Insert the root's index into a new heap.
  • Repeat k times: Remove the new heap's min element, go to the corresponding element of the original heap, and insert its two children into the new heap. The kth element that you remove from the new heap will be the kth smallest element.
