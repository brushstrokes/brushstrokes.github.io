In what order will the elements of the tree be visited in each type of traversal? (in-order, pre-order, post-order, level-order)
in-order: 4, 6, 8, 10, 15, 18, 21
Each element is visited after any element in its left subtree and before any element in its right subtree.

pre-order: 10, 6, 4, 8, 18, 15, 21
Each element is visited before any of its descendents.

post-order: 4, 8, 6, 15, 21, 18, 10
Each element is visited after any of its descendents.

level-order: 10, 6, 18, 4, 8, 15, 21
Each level is printed from left to right, starting with the top level and ending with the bottom level.

What is the worst-case runtime of an in-order traversal of an entire tree tree? You may assume that the "visit" operation on each node takes constant time. Give your answer in Θ notation as a function of n (the number of nodes).
Θ(n). The method is called once for each node of the tree, and there are a constant number of operations within each method call.
