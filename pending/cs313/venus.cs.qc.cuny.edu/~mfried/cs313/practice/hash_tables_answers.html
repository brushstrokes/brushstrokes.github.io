Hash tables questions

What is the average-case runtime of inserting an element into a hash table (using separate chaining)? Give your answer in Θ notation as a function of n (the number of elements). Assume that the number of buckets is always more than n.
Θ(1)

In what situation would you want to use a binary search tree instead of a hash table to implement a set?
If you want to iterate through the elements in sorted order.

Which is NOT an important property of a good hash function?
A. Equal keys must always have the same hash
B. Unequal keys must never have the same hash
C. The hashes should be uniformly distributed
D. The hashes should be efficient to compute
Unequal keys must never have the same hash.

Is this a good hashCode method? Explain your answer. 
public int hashCode() {
   return new Random().nextInt(); // Returns a random 32-bit integer
}
No because it is not consistent. Equal keys must always have the same hash.

If a hash table has 10 buckets, and every bucket contains exactly 5 keys, what is the load factor?
5. The load factor is the average number of keys per bucket.

If you want your hash table to allow load factors greater than 1, would you use separate chaining or open addressing? Explain your answer.
Separate chaining. In open addressing, the elements are stored in the array itself, so each bucket contains at most 1 element.

When implementing linear probing, when we delete an entry, we mark the slot as "deleted" rather than empty. Explain why.
If we mark an element in the middle of a cluster as empty, there may be elements in the right portion of the cluster that we will not be able to find when we search.
