What is the worst-case runtime of inserting an element into a binary search tree? Give your answer in Θ notation as a function of n (the number of elements).
Θ(n). The runtime depends on the height of the tree, which is Θ(n) in the worst case.

For the questions below, the Node class has three fields:
E data;
Node<E> left;

Write a method
boolean contains(E e, Node<E> root)
that takes the root of a binary search tree as a parameter and returns true if the tree contains element e.
You can do it either recursively or non-recursively. Full answer: BSTreeSet (non-recursive BSTreeSet)

Write a method
E minimum(Node<E> root)
that takes the root of a binary search tree as a parameter and returns the minimum element.
Repeatedly follow the left pointer as many times as you can, then return the node's data.

Write a method
int height(Node<E> root)
that takes the root of a binary search tree as a parameter and returns the height of the tree. The height is the number of edges in the longest path from root to leaf (a tree with only 1 node should have a height of 0, and an empty tree should have a height of -1).
First handle the base case, then calculate the height recursively: 1 + (the larger height of the two subtrees)

Write a method
boolean sameTree(Node<E> root1, Node<E> root2)
that takes the roots of two binary search trees as parameters and returns true if the trees are the same. Two trees are the same if they have exactly the same structure and values.
Base case: If both trees are empty, return true. If one is empty and one is not empty, return false.
Recursive case: Two trees are the same if their roots have the same data, and their left subtrees are the same, and their right subtrees are the same.

Write a method
boolean isBST(Node<E> root)
that takes the root of a binary tree as a parameter and returns true if it is a valid binary search tree. Hint: Do a traversal.
Do an in-order traversal to copy all elements into a List. Then check if the list is in increasing order.
Alternate answer: Keep track of the previous element as you do an in-order traversal. Check if each element is greater than the previous element.
