Inheritance

A subclass inherits the fields and methods of its superclass, which allows code to be reused.
The subclass may override some of the methods by having its own method with the same signature.
The subclass may also have additional fields and methods.
Inheritance is used when two classes have an "is a" relationship.

public class Person { String name; // The name field is inherited by Student void foo() {...} ... } public class Student extends Person { // Student is a subclass of Person double gpa; // Student has an additional field, gpa void foo() {...} // Overrides Person's foo method ... }

Abstract classes

An abstract class is a class that cannot be instantiated, but can be subclassed.
It may contain abstract methods (a method without an implementation).
A non-abstract subclass must implement all of the abstract methods in its superclass.

Interfaces

An interface is a collection of methods without method bodies.
A class may implement an interface by implementing all of the interface's methods.

Abstract classes vs interfaces

A class may only extend one superclass, but may implement multiple interfaces. An interface may also extend other interfaces.
Interfaces are commonly used to define types. For example, ArrayList implements List in the JCF.
An interface is like a contract that specifies what the code is supposed to do, so other programmers can use Lists in their code without knowing how the List is implemented. A programmer may switch from ArrayList to LinkedList, and the code will still work.
Interfaces can also be used to declare that a class provides some behavior. For example, if a class implements Comparable, then instances of the class can be compared.
Abstract classes may be used to define skeletal implementations for an interface, which makes it easier to implement the interface (e.g. AbstractList in the JCF).

Generics

We can create a generic class or interface that works with any type. For example, we can write one ArrayList class, and then we can create an ArrayList of Integers, Doubles, or Strings.

public interface List<E> { // E is a type parameter ... boolean add(E e); // Any class that implements List must implement add } public class ArrayList<E> implements List<E> { // E is a type parameter ... public boolean add(E e) {...} // We can use E as the element type }

When creating an ArrayList, specify the type inside < >. If the type is Integer, imagine that all Es in the class definition are replaced with Integer.
The type cannot be a primitive, so we must use the Integer wrapper class.
An int can be automatically converted to an Integer. This is called autoboxing.
Creating a generic array is not allowed, but we can create an Object array and cast (see sample code).

ArrayList<Integer> x = new ArrayList<>(); // Each element is an Integer x.add(5); // Since x is a list of Integers, this method takes an Integer as a parameter // An interface can be used as reference type, so we can also write: List<Integer> y = new ArrayList<>();

Exceptions

An exception is an object that gets thrown when an unexpected event occurs during a program's execution.
A method may handle an exception with a try-catch block, or throw the exception so another method can handle it.
If the exception is not handled, the program terminates and prints an error message to the console.
When an exception is thrown from inside a try block, execution skips ahead to the catch block.
Exceptions may be checked or unchecked. Code that may throw a checked exception must be enclosed by either a try block or a method with a throws clause, which specifies that it can throw an exception.

// Handling the exception within the same method public void foo() { try { if (...) throw new Exception(); // Code that may throw an exception } catch (Exception e) { // Code to handle the exception (executed only if an Exception was thrown) } }

// Handling the exception in another method public void foo1() { try { foo2(); // foo2 may throw an exception } catch (Exception e) { // Code to handle the exception (executed only if an Exception was thrown) } } public void foo2() throws Exception { // Specify that the method can throw an exception if (...) throw new Exception(); // Code that may throw an exception }

Nested classes

A nested class is a class defined inside another class. If a class is only used by one other class, making it nested can make the code more organized and increase encapsulation. For example, a Node class that is only used by a LinkedList class should be nested and private, because other classes should not be able to access it.
A nested class may either be static or nonstatic. An instance of a nonstatic nested class (aka an inner class) can only exist within an instance of the outer class, and has access to that object's fields and methods.
